# 萌芽期（第1-3轮）

> 📅 时间跨度：2011年 - 2012年
> 🎯 核心主题：生存与差异化——在资源最少的时候，价值观是唯一的指南针
> 
> **价值观①：做对用户有价值的事情**
> **价值观④：学习和快速迭代比经验更重要**

---

## 📍 第1轮 | 2011年上 | 「按住说话」| 重大决策

### 价值观冲突：**学习迭代 vs 经验跟随**

### 背景叙事

> 💬 **系统旁白**（逐行显示，打字机效果）：

> *2011年春节，南方通讯大厦六楼。*
> *13个人挤在一间会议室里。*
> *这里有一面很大的落地窗，面朝中山大道——*
> *但大家叫它「小黑屋」。*
> *不是因为黑，是因为赶起进度来没日没夜。*

#### 场景一：凌晨的小黑屋

你坐在Harry对面，屏幕上是微信后台的实时数据——日新增注册5000人。旁边的桌上堆着二十多部测试手机：诺基亚、黑莓、安卓、iPhone，围成一圈。Kiro埋着头在联调安卓客户端，你喊了一声「搞定了吗」，五分钟后对面传来一句「OK了」，头都没抬。

凌晨两点，Wawa端着一摞桂林米粉推门进来，米粉上码着他自己煎的蛋——隔壁小吃街只剩这一家还开着。你接过来，筷子还没动，手机响了。

**Alan**（语音消息，声音很平静）：

> *「你看一下Talkbox。」*

你打开App Store——一个来自香港的语音应用，3天100万下载。用户把自己录的语音转发到微博，像发明了新玩具。

你心里清楚：微信1.0只有文字和图片。你们是公司内三个类似项目里最小的一个。而外面，Talkbox在跑，米聊在跑，Kakao Talk也在跑。

**Justin**（把手机递给你，屏幕上是米聊的新版预告）：

> *「米聊下个版本要上语音对讲了。他们动作很快。」*

**Harry**（放下米粉）：

> *「语音对讲这事，不只是体验问题。摁住按钮发语音，这种交互在电脑上不存在，但在手机上非常自然。关键不在于提升通讯效率——在于它足够独特、好玩。」*

#### 场景二：Allen的选择题

第二天中午十一点，大家挤进Alan的办公室。他在白板上画了一张图：

```
┌──────────────────────────────────────┐
│         移动IM的「对的事情」           │
│                                      │
│    Talkbox ─── 语音 ─── 米聊         │
│                 ↓                    │
│            我们做不做？               │
│            做的话，怎么做？            │
└──────────────────────────────────────┘
```

**Alan**（转过身）：

> *「做语音这件事本身不难。难的是，我们到底要做一个跟Talkbox一样的东西，还是做一个不一样的东西。」*

他顿了顿。

> *「Lyle说了一句话我觉得很对：正确的东西做出来之后，一定会慢慢成长起来。但如果某些东西没做对，那种逆转是一定不会自然发生的。我们现在要找的是那件『对的事情』。」*

窗外，中山大道车流不息。会议室里很安静。

你需要做出选择。

### 选项设计

#### 选项A：快速跟进——上线语音消息（录完发送）

> *米聊马上就要发布语音对讲了，我们不能等。先做一个「录完发送」的语音消息，快速上线，抢回用户注意力。*

| 即时效果 | 延迟效果（1轮后） |
|---------|-----------------|
| 🔋 资源 -10 | 👥 用户 +50万 |
| 👨‍💻 团队 -5（加班赶工） | ⭐ 口碑 -5（与竞品同质化，媒体评价"又一个Talkbox"） |

#### 选项B：找到「对的事情」——做「按住说话」的语音对讲

> *不只是发语音。要做到「摁住、说话、松开、送达」一气呵成。语音按钮要像真的对讲机——带磨损金属的质感。接收端自动下载，点一下就能播。*

| 即时效果 | 延迟效果（2轮后） |
|---------|-----------------|
| 🔋 资源 -20（投入更大） | 👥 用户 +200万（后来居上） |
| 👨‍💻 团队 -10（高强度攻坚） | 💡 创新 +15（差异化壁垒） |
| | ⭐ 口碑 +10（"最爽的语音体验"） |

#### 选项C：不做语音——深耕文字和图片

> *语音是热点，但热点会过去。我们的定位是「免费短信」，把文字体验打磨到极致才是正道。*

| 即时效果 | 延迟效果（1轮后） |
|---------|-----------------|
| 🔋 资源 -5 | 👥 用户 +10万（增长乏力） |
| | ⭐ 口碑 +10（文字体验好评） |
| | 💡 创新 -10（错过语音窗口期） |

> ⚠️ **戏谑成就提示**：选C将解锁隐藏成就「🛋️ 南通6楼的幽灵」—— *"恭喜，你可能发明了另一个邮箱。"*

### 选择后叙事

#### 选择A后：「快，但不够爽」

你决定快速上线语音消息。团队用一个星期赶出了「录完发送」版本。上线当天，微信终于有了语音功能。

但你很快发现问题：用户每次要点「录制」→ 等待 → 点「发送」，流程冗长。米聊的用户说"两边都差不多"，媒体写了一篇《微信、米聊、Talkbox：移动语音IM三国杀》，把你们放在一起比较。

**一条微博**：
> @数码发烧友007：*试了下微信的语音，和米聊差不多嘛。都是录完发送，没啥区别。*

**Justin**（皱眉）：
> *「我们省了两周时间，但丢掉了差异化。」*

```
📊 延迟效果（1轮后生效）：
  👥 用户 +50万 ← 语音功能带来增长
  ⭐ 口碑 -5   ← "又一个Talkbox"
```

> 💡 **系统反思**：Alan后来说过："需求只来自你对用户的了解，不来自调研、分析、讨论、竞争对手。" 跟随竞品是最快的，但也最容易失去自己。

---

#### 选择B后：「杀米会」

你决定做「按住说话」。Alan的要求很简单，也很苛刻：

**Alan**：
> *「做到一个体验——用户默认就帮他下载好语音，他点一下就能播。不要让他等。」*

**Kink**（负责设计，回来后一脸为难）：
> *「他说语音按钮要做成一种磨损金属的质感，中间磨损得最厉害。就像真的对讲机一样。」*

**Kiro**（安卓开发，兴奋地插话）：
> *「Harvey提了一个方案——边发边录。发送的时候不等录完，一边录一边往服务器传。这样语音气泡就是秒发的。安卓上要引入一种全新的编解码技术，但做出来真的能给用户省流量。」*

五一假期，米聊抢先上线了语音对讲。米聊的用户数一度反超微信。微博上到处是米聊的讨论。

Justin把团队召集起来——

**Justin**（站在白板前）：
> *「『杀米会』。五一不放假了。我们留下来把2.0做完。」*

谁也没有怨言。你知道为什么：因为每个人都把自己投入进去了。

5月10日，微信2.0上线。扬声器播放语音，手机贴近耳朵自动切听筒。语音气泡「秒发」。连续多条语音自动播放，像电台一样，解放双手。未读语音的红点提醒不会自动消失——

**Wawa**：
> *「Allen一直强调，要让用户有一种爽的感觉。米聊的语音要自己点下载，下载完才能播。我们是默认先帮你下好。一个小小的差异，交流的间隔感完全不同。」*

5月19日凌晨00:49，Alan发了一条微博：

> *「辛苦了很久，微信的同学们今晚享受到用户暴涨的喜悦了。Mark一下。」*

微信的用户数重新盖过米聊，日新增从一万飙升到五万。

```
📊 延迟效果（2轮后生效）：
  👥 用户 +200万 ← 语音对讲后来居上
  💡 创新 +15   ← "边发边录"技术壁垒
  ⭐ 口碑 +10   ← "最爽的语音体验"
```

> 💡 **系统反思**：
> **价值观④：学习和快速迭代比经验更重要**
> 微信没有跟着竞品"抄作业"，而是从用户的真实体验出发——「点一下就能播」「贴近耳朵自动切听筒」「边发边录秒送达」——每一个细节都在回答同一个问题：*用户用起来爽不爽？*
> Lake在旁边见证了2.0发布的那一刻，他说："我们三三两两围在一起，等待产品『发射』。因为每个人都把自己投入进去了，只有投入之后才那么关心它，像关心一个自己即将出生的孩子一样。"

---

#### 选择C后：「南通6楼的幽灵」

你决定不做语音。团队继续在文字和图片体验上精耕细作——每条消息的排版、已读未读的设计、对话气泡的视觉细节……

三月过去了。

米聊上了语音对讲，用户数翻了三倍。Talkbox登上App Store排行榜首位。而微信的日新增还在5000人左右徘徊。

Bony让秘书转来一句话：*「语音是趋势。你们怎么看？」*

微信成了公司里「那个不做语音的聊天工具」。

```
📊 延迟效果（1轮后生效）：
  👥 用户 +10万   ← 增长乏力
  ⭐ 口碑 +10     ← 文字体验好评
  💡 创新 -10     ← 错过窗口期
  🔋 资源 -10     ← 高层质疑，资源被削减
```

> 🏆 **成就解锁**：🛋️ **南通6楼的幽灵** —— *"你选择了不做语音。恭喜，你可能发明了另一个邮箱。"*

> 💡 **系统反思**：微信后来的七条价值观中写道：「学习和快速迭代比经验更重要——移动互联网变化太快，产品和业务思路要面向新环境产生。」不做语音不是"错误"，但在2011年，语音对讲是移动IM的结构性变化。错过结构性变化的窗口，后面再追就难了。

#### 场景一：夜会

语音对讲上线了，日新增从5000涨到了5万。但这还不够——手机QQ的日活已经过亿，公司高层开始有人说「微信是不是应该合并到手Q？」

南方通讯大厦楼层比较矮，不超过三米。晚上，十来个人挤在Alan的办公室，窗外的中山大道已经安静下来。

有人随口说了一句：「楼上说不定就有个美女在用微信，但我们不知道她在哪。」

空气突然安静了一秒。

**Justin**（眼睛亮了）：
> *「智能手机有GPS。如果能让用户看到附近谁在用微信——」*

**Harry**（接话）：
> *「用网格划分，一个格子一个格子查。技术上不难，两三天能做出来。」*

**Alan**（没有立刻回答。他在想别的事）：
> *「通讯软件第一重要的需求是什么？找到人。谁最先迈过用户密度这个坎，谁就赢了。」*

但你心里也隐隐有另一种担忧：让陌生人看到彼此的位置——这会不会打开一个潘多拉魔盒？

#### 场景二：三条路

你有三个方向可以选。

---

### 📎 历史线索

> **📎 线索一**（可选查看）
> 
> **类型**：引言
> **内容**："附近的人，给微信带来的最大价值就是勾起了用户的好奇心，给了他们一个来安装微信的理由。" —— Harry，接受媒体采访

---

### 选项设计

#### 选项A：做「附近的人」——用好奇心引爆增长

> *让用户看到附近谁也在用微信。GPS定位，网格算法，两三天就能上线。这是增长最猛的一剂药——但你知道，「附近的陌生人」这把钥匙，可能也会打开灰色地带的大门。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 🔋 资源 -15 | **1轮后**：👥 用户 +200万（自然安装量达百万级/天） |
| | **2轮后**：⭐ 口碑 -15（灰产问题爆发，媒体负面报道） |

> ⚠️ **分支触发**：选择A将在下一轮插入分支事件「灰产困境」

#### 选项B：做「摇一摇」——用创意创造惊喜

> *Harry在研究Bump的时候，摇了一下手机，竟然摇出一个真人。那种心动的交互感——如果能做到微信上？动作简单，反馈直观，利用手机重力感应，一个星期就能做完。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 🔋 资源 -20 | **1轮后**：👥 用户 +100万 |
| 💡 创新 +10 | ⭐ 口碑 +5（"好玩"） |

> ⚠️ **分支触发**：选择B将在下一轮插入分支事件「摇一摇的极简」

#### 选项C：优化QQ好友导入——借大树乘凉

> *微信不需要重新发明社交关系。QQ有十亿用户，做好好友导入，微信的关系链自然就有了。安全、稳妥、低成本。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 🔋 资源 -5 | **1轮后**：👥 用户 +80万 |
| 💡 创新 -10（跟随路线） | 被媒体和内部视为「QQ附属品」 |

> ⚠️ **分支触发**：选择C将在下一轮插入分支事件「摇一摇的极简」

### 选择后叙事

#### 选择A后：「附近有人」

你决定做「附近的人」。Harvey和Stephen两三天搞定了方案，找了实习生Junchen写完代码，测试一两天就上了。

2011年8月3日，微信2.5上线。

功能发布一秒钟——

**Lynncui**（兴奋地叫起来）：
> *「里面有个美女！快加快加！」*

整个办公室炸了。所有人疯狂地给她打招呼、加好友。

过了五分钟，Ts默默地说了一句：

**Ts**：
> *「……那是我老婆。」*

哄堂大笑。但笑声背后，数据已经说明了一切——微信每天的自然安装量达到了**百万级**。

Alan后来讲过一个故事：他向几个女同学推荐微信——"免费发短信，还能语音，省钱省力"——几个人无动于衷。直到他演示「附近的人」，她们两眼放光，当场安装。

**Alan**：

```

## 📍 分支B2 | 2011年下 | 「摇一摇的极简」

摇一摇的原型做出来了——内部代号叫「撸一撸」，服务器命名叫「撸server」。开发它的同事Andrewang被大家笑称为「撸神」。

**Kink**（尴尬地笑）：
> *「设计稿拿出来后，团队反馈说可能略有不雅。最后索性换成了最人畜无害的小花。」*

声音呢？一开始给女生用户设置的是「叮叮当当」。后来发现枪声男女都喜欢——因为它代表了刺激。于是统一成了来福枪上膛声。

#### 场景二：Alan的减法

**Alan**（转过身，语气很平）：
> *「这个东西，要么不做，要做就做到极简。」*

**Harry**：
> *「但竞品都有筛选功能，用户会不会觉得我们太简陋？」*

Alan没有直接回答。他走到白板前，画了四个字：

```
  动作 → 声音 → 画面 → 结果
  摇     咔嚓    开门    出人
```

**Alan**：
> *「这四步必须一气呵成。中间不能有任何多余的东西。」*
> 
> *「竞争对手一定会抄。但如果我们做到了极简，他们只能往上加东西。加东西就意味着变复杂，变复杂就输了。」*

他顿了顿，又补了一句：

> *「弗洛伊德说，性是人类最原始的驱动力。这个动作非常简单，又能让你很爽——最原始的东西，往往就是体验最好的。」*

你看着白板上的四个字。选择摆在面前。

---

### 选项设计

#### 选项A：极简路线——摇、咔嚓、开门、出人

> *砍掉所有附加功能。没有筛选，没有菜单，没有按钮。整个界面只有一张图，只需要一个动作。*

| 即时效果 | 延迟效果（1轮后） |
|---------|-----------------|
| ⭐ 口碑 +5 | 竞品全部抄了摇一摇，但无一超越 |
| 💡 创新 +5 | 👥 用户增长稳健（口碑传播） |

#### 选项B：功能丰富路线——加上筛选、模板、地图

> *用户要的是"找到对的人"。性别筛选、距离筛选、打招呼模板——这些功能竞品都有，我们没有理由不做。*

| 即时效果 | 延迟效果（1轮后） |
|---------|-----------------|
| 👥 用户 +5万（短期见效） | 💡 创新 -5（与竞品同质化） |

#### 选项C：折中路线——极简界面 + 后台筛选

> *界面极简，但后台默默加上距离和性别的匹配权重。用户看不到复杂性，但体验更精准。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 👥 用户 +4万 | 体验略有割裂 |
| 💡 创新 +3 | |

### 选择后叙事

#### 选择A后：「无法超越」

摇一摇上线了。界面里没有任何按钮和菜单——只有一张图。

上线第一天：千万级注册，**两亿次摇动**。

Bony给Alan发了一封邮件，大意是：摇一摇很好，但要防竞品抄。Alan回了一段话——后来成了产品史上的经典：

**Alan**：
> *「我们这个功能别人没法超越。一方面我们已经最简化，摇一摇只有一个动作；另一方面，摇一摇整个体验过程是在人类的性驱动力下完成的。功能很简单，又能让你很爽，因而无法被超越。」*

果然，竞品后来全部抄了摇一摇，但无一例外都加了额外元素——筛选、标签、花哨动画。反而显得臃肿。

```
📊 分支效果：
  ⭐ 口碑 +5  ← 极简体验
  💡 创新 +5  ← 差异化壁垒
```

> 💡 **系统反思**：
> **极简方能不被超越。**
> Alan在《微信背后的产品观》中说："我们把这个功能做到了极简化。如果一个东西已经做到非常非常少了，要更少是很困难的。"
> 做减法比做加法难得多。而最少的东西，最难被超越。

**Alan**（看着评论，沉默了很久）：
> *「我们本来可以做到最简的。」*

```
📊 分支效果：
  👥 用户 +5万  ← 短期见效
  💡 创新 -5    ← 与竞品同质化
```

#### 选择C后：「半透明的复杂」

摇一摇的界面很简洁，但你知道后台藏着距离和性别的匹配权重。上线几天后，有用户发现了：

## 📍 第3轮 | 2012年上 | 「朋友圈的方向」| 重大决策

### 价值观冲突：**系统思维 vs 流量思维**

### 背景叙事

> 💬 **系统旁白**：
> 
> *2012年3月。微信用户突破1亿。*
> *433天，从0到100,000,000。*
> *有人开了香槟，有人拉了横幅。*
> *但Alan坐在角落，想的是另一件事。*

---

#### 场景一：破亿之后

会议室的横幅上写着：「历时433天，从0到100,000,000用户」。桌上摆着QQ企鹅公仔，几个人在拍照。

但团队很快面临一个新问题：微信现在有1亿人了——他们打完字、发完语音之后，要去哪里？

微博如日中天。所有人都在刷信息流、转发、评论。社交媒体是大势所趋。

产品团队提出做"朋友圈"——但方向有分歧。

**晴仔**（朋友圈产品经理，推了一摞设计稿上来）：
> *「我们做了超过40个版本的设计方案。从A目录到M目录，从style_1到style_5……Alan全部看过了。」*

40多个版本。这不是在做一个功能，是在找一个方向。

#### 场景二：什么样的广场

**Alan**（在白板上画了两个圆）：
> *「左边是微博——一个开放广场。任何人都能看到任何人说的话。右边是私密相册——只有自己能看。」*
> 
> *「这两个极端之间，有没有第三种可能？」*

他在两个圆之间画了一条虚线。

> *「朋友圈本质上是一个社交广场。但它应该是一个有边界的广场——只有共同好友能看到你们的互动。」*

**晴仔**：
> *「但这样的话，信息流的内容量会比微博少很多。用户会不会觉得无聊？」*

**Alan**：
> *「如果一个解决方案非常复杂，一定是问题错了。我们不是要做一个内容很多的信息流——我们要做的是，让每一条内容都和你有关。」*

| 🔋 资源 -15 | **1轮后**：👥 用户 +300万（信息流带来高活跃） |
| | ⭐ 口碑 -10（信息过载，沦为"微博克隆"） |

#### 选项B：做纯私密相册

> *极致的私密性。只有自己能看到自己发的内容，像一本数字日记。质量很高，但增长受限。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 🔋 资源 -10 | **1轮后**：⭐ 口碑 +15（体验精致） |
| | 👥 用户 +100万（增长受限） |

#### 选项C：做半私密社交圈（反复打磨40+版本）

> *一个有边界的广场：你发的内容，只有你的好友能看到。好友的评论，只有共同好友能看到。每一条内容都和你有关——没有噪音，只有"自己人"的交流。代价是——要打磨很久很久。*

| 即时效果 | 延迟效果 |
|---------|---------|
| 🔋 资源 -25（投入极大） | **2轮后**：👥 用户 +250万 |
| 💡 创新 +15 | ⭐ 口碑 +20（与微博形成关键差异化） |
| | 💡 创新 +10（系统思维红利） |

### 选择后叙事

#### 选择A后：「第二个微博」

朋友圈做成了开放广场。上线一周，数据很亮眼——转发量、浏览量都很高。

但很快，你开始在内部群里看到这样的声音：

> @产品分析师李明：*微信朋友圈和微博有啥区别？一模一样的信息流。腾讯又做了一个「跟随者」。*

用户开始在朋友圈里看到大量不认识的人的内容、广告、转发链条。刷朋友圈变成了一件有负担的事。

```
📊 延迟效果（1轮后生效）：
  👥 用户 +300万 ← 开放信息流高活跃
  ⭐ 口碑 -10   ← "微博克隆"
```

#### 选择B后：「完美的孤岛」

朋友圈做成了私密相册。体验非常精致，每个细节都打磨到位。

但用户打开朋友圈，发现只有自己发的内容。

**一条反馈**：
> *朋友圈挺好看的，但就是……有点孤独。*

```
📊 延迟效果（1轮后生效）：
  ⭐ 口碑 +15   ← 体验精致
  👥 用户 +100万 ← 增长受限
```

#### 选择C后：「有边界的广场」

40多个版本的打磨。晴仔带着团队反复迭代——A方案到M方案，style_1到style_5。

Alan亲自回复每一条内测用户的反馈。

终于上线的那天，朋友圈看起来很简单——刷信息流，点赞，评论。但底层逻辑完全不同于微博：

- 你只能看到好友发的内容
- 好友的评论，只有共同好友能看到
- 没有"转发"按钮——每一条内容都是原创的

用户一开始觉得"怎么内容这么少"。但很快，他们发现了一个奇妙的体验——**在朋友圈看到的每一条内容，都和自己有关**。没有噪音，没有陌生人。

**用户微博**：
> @一个普通用户：*微博刷半小时感觉被掏空。朋友圈刷十分钟，感觉和朋友聊了天。这个区别，说不上来但很明显。*

```
📊 延迟效果（2轮后生效）：
  👥 用户 +250万 ← 口碑传播带来的有机增长
  ⭐ 口碑 +20   ← 与微博形成关键差异化
  💡 创新 +10   ← 系统思维红利
```

> 💡 **系统反思**：
> **价值观⑤：系统思维——建造透明公正的自运转系统，避免人为干预**
> Alan说过："用户是人，产品经理应像上帝一样建立系统、制定规则，让群体自我演化。"朋友圈的"半私密"设计就是一个系统——不是靠运营去筛选内容，而是用规则（只有共同好友可见）让好内容自然浮现。
> 微博是广播，朋友圈是对话。这不是技术差异，是价值观差异。
